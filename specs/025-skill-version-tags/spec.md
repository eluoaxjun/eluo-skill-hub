# Feature Specification: 스킬 버전 관리 및 태그 기능

**Feature Branch**: `025-skill-version-tags`
**Created**: 2026-03-09
**Status**: Draft
**Input**: User description: "스킬 추가하기에서 기능 추가 - 버전 관리, 태그 추가 (#마케팅, #GEO 등), 수정하기 동일 동작"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스킬에 버전 번호 지정 (Priority: P1)

관리자가 스킬을 추가하거나 수정할 때 버전 번호(예: 1.0.0)를 입력하여 스킬의 변경 이력을 관리할 수 있다. 대시보드 사용자는 현재 버전 정보를 확인할 수 있다.

**Why this priority**: 버전 관리는 스킬 업데이트 추적의 핵심 기능이며, 사용자가 최신 스킬을 식별하는 데 필수적이다.

**Independent Test**: 스킬 추가 폼에서 버전 번호를 입력하고 저장한 뒤, 스킬 상세에서 해당 버전이 표시되는지 확인하여 독립 테스트 가능

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 추가 폼에 있을 때, **When** 버전 번호(예: 1.0.0)를 입력하고 저장하면, **Then** 해당 스킬에 입력한 버전 번호가 저장된다
2. **Given** 스킬이 이미 존재할 때, **When** 관리자가 수정 폼에서 버전 번호를 변경하면, **Then** 이전 버전은 변경 이력에 기록되고 새 버전으로 업데이트된다
3. **Given** 스킬이 버전 정보를 가지고 있을 때, **When** 사용자가 스킬 상세를 열면, **Then** 현재 버전 번호가 표시된다

---

### User Story 2 - 스킬에 태그 추가 및 관리 (Priority: P1)

관리자가 스킬을 추가하거나 수정할 때 태그(예: #마케팅, #GEO, #퍼블리싱)를 자유롭게 입력하여 분류 및 검색을 돕는다.

**Why this priority**: 태그는 카테고리보다 세밀한 분류를 제공하며, 사용자의 스킬 탐색 경험을 크게 향상시킨다.

**Independent Test**: 스킬 추가 폼에서 태그를 입력하고 저장한 뒤, 스킬 카드 및 상세에서 태그가 표시되는지 확인하여 독립 테스트 가능

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 추가 폼에 있을 때, **When** 태그 입력란에 "마케팅"을 입력하고 Enter 또는 쉼표를 누르면, **Then** 태그가 칩(chip) 형태로 추가된다
2. **Given** 태그가 이미 추가된 상태에서, **When** 관리자가 태그 칩의 삭제 버튼을 누르면, **Then** 해당 태그가 제거된다
3. **Given** 스킬에 태그가 지정된 상태에서, **When** 사용자가 스킬 카드 또는 상세 팝업을 보면, **Then** 해당 태그들이 표시된다

---

### User Story 3 - 버전 변경 이력 조회 (Priority: P2)

관리자가 스킬 수정 시 이전 버전 이력을 확인할 수 있어, 어떤 변경이 언제 이루어졌는지 추적할 수 있다.

**Why this priority**: 버전 이력은 관리 편의성을 높이지만 핵심 기능(버전 지정, 태그)보다는 부가적이다.

**Independent Test**: 스킬을 여러 번 수정하며 버전을 올린 뒤, 수정 폼에서 이전 버전 목록이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 스킬이 여러 번 수정되어 버전 이력이 있을 때, **When** 관리자가 수정 폼을 열면, **Then** 이전 버전 번호와 변경 일시 목록이 표시된다
2. **Given** 스킬이 처음 생성된 상태일 때, **When** 관리자가 수정 폼을 열면, **Then** 현재 버전만 표시되고 이력은 비어있다

---

### User Story 4 - 태그 기반 스킬 필터링 (Priority: P3)

대시보드 사용자가 태그를 클릭하여 해당 태그가 지정된 스킬만 필터링할 수 있다.

**Why this priority**: 탐색 편의 기능으로, 태그 추가/표시 기능이 먼저 완성된 후 구현 가능하다.

**Independent Test**: 대시보드에서 스킬 카드의 태그를 클릭한 뒤, 동일 태그를 가진 스킬만 필터링되는지 확인

**Acceptance Scenarios**:

1. **Given** 대시보드에 여러 태그가 지정된 스킬이 있을 때, **When** 사용자가 특정 태그를 클릭하면, **Then** 해당 태그를 가진 스킬만 표시된다

---

### Edge Cases

- 버전 번호를 입력하지 않고 저장하면 기본값 "1.0.0"이 자동 지정된다
- 태그를 하나도 입력하지 않아도 스킬 저장이 가능하다 (태그는 선택사항)
- 동일한 태그명을 중복 입력하면 하나만 추가된다 (대소문자 무시)
- 태그 입력 시 '#' 접두사는 자동으로 제거하여 저장한다 (표시할 때만 '#' 붙임)
- 태그명의 앞뒤 공백은 자동으로 제거된다
- 빈 문자열 태그는 무시된다
- 버전 번호 형식은 시맨틱 버저닝(X.Y.Z) 형식을 권장하되, 자유 형식 텍스트도 허용한다 (예: "2.0", "v3", "beta")
- 기존 스킬(DB에 이미 존재하는 스킬)의 버전 번호 기본값은 "1.0.0"으로 마이그레이션된다
- 태그 하나의 최대 길이는 30자로 제한한다
- 스킬당 최대 태그 수는 10개로 제한한다
- 버전 번호의 최대 길이는 20자로 제한한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 스킬 추가 시 버전 번호 입력 필드를 제공해야 한다
- **FR-002**: 시스템은 스킬 수정 시 버전 번호를 변경할 수 있어야 한다
- **FR-003**: 시스템은 버전이 변경될 때마다 이전 버전 정보(버전 번호, 변경 일시)를 이력으로 보관해야 한다
- **FR-004**: 시스템은 스킬 추가 시 태그 입력 기능을 제공해야 한다
- **FR-005**: 시스템은 스킬 수정 시 기존 태그를 추가/제거할 수 있어야 한다
- **FR-006**: 태그 입력은 Enter 키 또는 쉼표(,)로 구분하여 추가할 수 있어야 한다
- **FR-007**: 추가된 태그는 칩(chip) UI로 표시되며 개별 삭제가 가능해야 한다
- **FR-008**: 시스템은 태그명 중복을 자동으로 방지해야 한다 (대소문자 무시)
- **FR-009**: 스킬 상세 팝업에서 현재 버전 번호와 태그 목록이 표시되어야 한다
- **FR-010**: 대시보드 스킬 카드에 태그가 표시되어야 한다
- **FR-011**: 관리자 스킬 수정 폼에서 버전 변경 이력을 조회할 수 있어야 한다
- **FR-012**: 버전 번호 미입력 시 기본값 "1.0.0"이 자동 적용되어야 한다
- **FR-013**: 태그는 선택사항이며, 태그 없이도 스킬 저장이 가능해야 한다
- **FR-014**: 대시보드에서 태그를 클릭하면 해당 태그를 가진 스킬이 필터링되어야 한다

### Key Entities

- **버전 이력(Version History)**: 스킬의 버전 변경 기록. 스킬 ID, 버전 번호, 변경 일시, 변경 메모(선택)를 포함한다. 하나의 스킬에 여러 버전 이력이 존재할 수 있다.
- **태그(Tag)**: 스킬에 부여되는 자유 형식 키워드. 태그명은 고유하며, 여러 스킬이 동일 태그를 공유할 수 있다(다대다 관계). 최대 30자, 스킬당 최대 10개.

## Assumptions

- 버전 번호는 자유 형식 텍스트로 저장한다 (시맨틱 버저닝을 권장하되 강제하지 않음)
- 태그는 전역 풀로 관리한다 (같은 이름의 태그는 하나만 존재, 여러 스킬에서 공유)
- 기존 skills 테이블에 `version` 컬럼을 추가하고, 별도의 버전 이력 테이블로 이력을 관리한다
- 태그는 태그 테이블과 스킬-태그 조인 테이블로 관리한다 (다대다 관계)
- 기존 스킬 데이터는 마이그레이션 시 version "1.0.0"으로 설정한다
- 관리자만 태그와 버전을 설정할 수 있다 (기존 관리자 권한 체계 활용)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 스킬 추가/수정 시 버전 번호와 태그를 30초 이내에 입력 완료할 수 있다
- **SC-002**: 스킬 상세 팝업에서 버전 정보와 태그가 1초 이내에 로드된다
- **SC-003**: 태그 필터링 시 결과가 1초 이내에 표시된다
- **SC-004**: 기존 스킬 데이터가 마이그레이션 후 정상적으로 버전 "1.0.0"과 함께 표시된다
- **SC-005**: 스킬 수정 폼에서 이전 버전 이력이 정확하게 시간순으로 표시된다
