# Feature Specification: 어드민 스킬관리 페이지 디자인

**Feature Branch**: `010-admin-skill-management`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "스킬관리 페이지 디자인 - stitch-html/admin-skills.html의 Scrollable Content를 참고하며, 검색기능 추가"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스킬 목록 카드 그리드 조회 (Priority: P1)

관리자가 스킬 관리 페이지에 진입하면, 등록된 모든 스킬을 카드 형태의 그리드 레이아웃으로 한눈에 확인할 수 있다. 각 카드에는 스킬 아이콘, 상태 뱃지(Published/Draft), 스킬명, 설명, 카테고리가 표시되며, 수정/삭제 버튼이 포함된다. 기존 테이블 레이아웃을 카드 그리드로 전환하여 시각적 직관성을 높인다.

**Why this priority**: 스킬 관리의 핵심 화면으로, 모든 후속 기능(검색, 필터)의 기반이 된다.

**Independent Test**: 스킬 관리 페이지 진입 시 등록된 스킬들이 카드 그리드로 표시되고, 각 카드의 정보(아이콘, 상태, 이름, 설명, 카테고리, 액션 버튼)가 올바르게 렌더링되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 등록된 스킬이 5개 있는 상태에서, **When** 관리자가 스킬 관리 페이지에 진입하면, **Then** 5개의 스킬 카드가 그리드 형태(데스크탑 3열, 태블릿 2열, 모바일 1열)로 표시된다.
2. **Given** 스킬 카드가 표시된 상태에서, **When** 각 카드를 확인하면, **Then** 스킬 아이콘, 상태 뱃지(Published 또는 Draft), 스킬명, 설명(최대 2줄), 카테고리, 수정/삭제 버튼이 모두 표시된다.
3. **Given** 등록된 스킬이 없는 상태에서, **When** 관리자가 스킬 관리 페이지에 진입하면, **Then** "등록된 스킬이 없습니다" 안내 메시지가 표시된다.
4. **Given** 스킬 카드 그리드 마지막에, **When** 그리드를 확인하면, **Then** "새 스킬 추가하기" 플레이스홀더 카드가 점선 테두리로 표시된다.

### User Story 2 - 스킬 검색 (Priority: P1)

관리자가 스킬 관리 페이지 상단의 검색 입력란에 키워드를 입력하면, 스킬명 또는 설명에 해당 키워드가 포함된 스킬만 실시간으로 필터링되어 카드 그리드에 표시된다. 기존 회원관리 페이지의 검색 패턴(디바운스 적용)을 따른다.

**Why this priority**: 스킬 수가 증가하면 원하는 스킬을 빠르게 찾는 것이 관리자 효율에 직접적으로 영향을 미친다.

**Independent Test**: 검색란에 키워드를 입력하면 해당 키워드와 매칭되는 스킬만 카드 그리드에 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 스킬 관리 페이지에 진입한 상태에서, **When** 검색란에 "고객"을 입력하면, **Then** 스킬명 또는 설명에 "고객"이 포함된 스킬 카드만 표시된다.
2. **Given** 검색란에 키워드를 입력하는 중에, **When** 타이핑이 잠시 멈추면(디바운스), **Then** 검색 결과가 반영된다.
3. **Given** 검색 결과가 0건인 경우, **When** 결과 영역을 확인하면, **Then** "검색 결과가 없습니다" 안내 메시지가 표시된다.
4. **Given** 검색란에 키워드가 입력된 상태에서, **When** 검색란을 비우면, **Then** 전체 스킬 목록이 다시 표시된다.

---

### User Story 3 - 상태별 필터 탭 (Priority: P2)

관리자가 페이지 상단의 필터 탭(전체/배포됨/초안)을 클릭하면, 선택한 상태에 해당하는 스킬만 카드 그리드에 표시된다. 활성 탭은 시각적으로 구분된다.

**Why this priority**: HTML 디자인 레퍼런스에 포함된 기능으로, 상태별 스킬 파악을 빠르게 지원한다.

**Independent Test**: 각 필터 탭 클릭 시 해당 상태의 스킬만 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 스킬 관리 페이지에서 "전체" 탭이 활성화된 상태에서, **When** "배포됨" 탭을 클릭하면, **Then** 상태가 Published인 스킬 카드만 표시된다.
2. **Given** "초안" 탭이 활성화된 상태에서, **When** "전체" 탭을 클릭하면, **Then** 모든 스킬 카드가 표시된다.
3. **Given** 필터 탭과 검색이 모두 적용된 상태에서, **When** 결과를 확인하면, **Then** 두 조건을 모두 만족하는 스킬만 표시된다.

---

### User Story 4 - 페이지 헤더 영역 (Priority: P2)

스킬 관리 페이지 상단에 "스킬 관리" 제목과 부제("기업 내 등록된 모든 AI 스킬 및 에이전트 현황입니다.")가 표시되며, HTML 레퍼런스의 레이아웃을 따른다. 제목 영역과 필터 탭이 양쪽으로 배치된다.

**Why this priority**: 페이지 정체성을 나타내는 기본 UI 요소이다.

**Independent Test**: 페이지 진입 시 제목, 부제, 필터 탭이 올바른 위치에 렌더링되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 관리 페이지에 진입하면, **When** 상단 영역을 확인하면, **Then** 좌측에 "스킬 관리" 제목과 부제가, 우측에 필터 탭이 표시된다.

---

### Edge Cases

- 스킬 설명이 매우 긴 경우 카드 내에서 최대 2줄로 잘려야 한다.
- 스킬 수가 많을 경우 기존 페이지네이션이 유지되어야 한다.
- 검색어에 특수문자가 포함된 경우에도 오류 없이 동작해야 한다.
- 필터 탭 변경 시 검색어가 유지되어야 한다.
- 모바일에서 카드 그리드가 1열로 반응형 전환되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 등록된 스킬을 카드 형태의 그리드 레이아웃으로 표시해야 한다 (데스크탑 3열, 태블릿 2열, 모바일 1열).
- **FR-002**: 각 스킬 카드에는 스킬 아이콘, 상태 뱃지(Published/Draft), 스킬명, 설명(최대 2줄 말줄임), 카테고리, 수정 버튼, 삭제 버튼이 표시되어야 한다.
- **FR-003**: 카드 그리드 마지막에 "새 스킬 추가하기" 플레이스홀더 카드가 점선 테두리로 표시되어야 한다.
- **FR-004**: 시스템은 검색 입력란을 제공하여 관리자가 스킬명 또는 설명 기준으로 스킬을 검색할 수 있어야 한다.
- **FR-005**: 검색은 디바운스가 적용되어 입력 중 불필요한 요청을 방지해야 한다.
- **FR-006**: 검색 결과가 없을 때 "검색 결과가 없습니다" 안내 메시지를 표시해야 한다.
- **FR-007**: 시스템은 상태별 필터 탭(전체/배포됨/초안)을 제공하여 스킬을 상태별로 필터링할 수 있어야 한다.
- **FR-008**: 필터 탭과 검색은 동시에 적용 가능해야 하며, 두 조건을 모두 만족하는 스킬만 표시되어야 한다.
- **FR-009**: 페이지 상단에 "스킬 관리" 제목, 부제, 필터 탭이 HTML 레퍼런스의 레이아웃에 맞게 배치되어야 한다.
- **FR-010**: 기존 페이지네이션은 유지되어야 하며, 필터/검색 조건 변경 시 첫 페이지로 초기화되어야 한다.
- **FR-011**: 카드에 호버 시 미세한 상승 애니메이션과 그림자 효과가 적용되어야 한다.
- **FR-012**: 어드민 사이드바 내비게이션 항목의 폰트 크기를 16px(text-base)로 변경해야 한다. (공유 컴포넌트 — 모든 어드민 페이지에 적용)

### Key Entities

- **Skill(스킬)**: 스킬명, 설명, 카테고리, 상태(Published/Draft), 아이콘, 등록일
- **Filter(필터)**: 상태값(전체/배포됨/초안), 검색 키워드

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 스킬 관리 페이지 진입 후 1초 이내에 스킬 카드 그리드가 렌더링된다.
- **SC-002**: 검색 입력 후 결과가 화면에 반영되기까지 0.5초 이내에 완료된다.
- **SC-003**: 필터 탭 전환 시 0.3초 이내에 해당 상태의 스킬만 표시된다.
- **SC-004**: 모바일, 태블릿, 데스크탑 모든 뷰포트에서 카드 레이아웃이 올바르게 반응형 전환된다.
- **SC-005**: 기존 테이블 레이아웃 대비 스킬 정보 파악 시간이 단축된다 (카드 레이아웃의 시각적 직관성 향상).

## Clarifications

### Session 2026-03-04

- Q: 어드민 사이드바 내비게이션 항목의 폰트 크기는? → A: 16px (text-base)로 변경. 기존 text-sm(14px)에서 증가. 사이드바는 공유 컴포넌트이므로 모든 어드민 페이지에 적용.

## Assumptions

- HTML 레퍼런스(`stitch-html/admin-skills.html`)의 Scrollable Content 영역 디자인을 충실히 따른다.
- 검색 디바운스는 기존 회원관리 검색과 동일한 방식(300ms)을 적용한다.
- "수정" 버튼 클릭 시의 동작(편집 페이지 이동 등)은 본 스펙 범위 밖이며, 버튼 UI만 구현한다.
- "삭제" 버튼 클릭 시의 동작(확인 다이얼로그 등)은 본 스펙 범위 밖이며, 버튼 UI만 구현한다.
- "새 스킬 추가하기" 카드 클릭 시의 동작은 본 스펙 범위 밖이며, 플레이스홀더 UI만 구현한다.
- 스킬 아이콘은 카테고리별로 기본 아이콘을 매핑하여 표시한다.
- glass-card 스타일(반투명 배경, backdrop-blur, 미세 그림자)은 HTML 레퍼런스를 참고한다.
