# Implementation Plan

- [x] 1. 기반 타입 정의 및 정적 데이터 준비
- [x] 1.1 대시보드 공유 타입 정의
  - 직군 카테고리 타입(기획/디자인/퍼블리싱/개발/QA)과 카테고리 선택 타입을 정의한다
  - 스킬 요약 정보 인터페이스를 정의한다 (스킬명, 설명, 카테고리, 태그)
  - 대시보드 상태 인터페이스와 액션 인터페이스를 정의한다
  - 카테고리 항목 인터페이스를 정의한다 (ID, 라벨, 아이콘 타입)
  - `any` 타입 사용 금지, `readonly` 속성으로 불변성을 보장한다
  - _Requirements: 2.2, 4.3_

- [x] 1.2 (P) 카테고리 정의 상수 생성
  - "전체" 포함 6개 카테고리(전체/기획/디자인/퍼블리싱/개발/QA) 상수 배열을 정의한다
  - 각 카테고리에 직군을 식별할 수 있는 아이콘을 매핑한다 (lucide-react 아이콘 사용)
  - `readonly` 배열로 불변성을 보장한다
  - lucide-react 패키지가 프로젝트에 설치되어 있지 않으면 의존성을 추가한다
  - _Requirements: 2.2, 2.3_

- [x] 1.3 (P) 목 스킬 데이터 생성
  - 5개 직군별로 최소 2개씩 총 10개 이상의 목 스킬 데이터를 생성한다
  - 각 스킬에 이름, 설명, 카테고리, 태그, 아이콘 정보를 포함한다
  - `readonly` 배열로 불변성을 보장하고, 정의된 스킬 요약 타입을 준수한다
  - _Requirements: 4.2, 4.3_

- [x] 2. 유틸리티 훅 구현
- [x] 2.1 미디어 쿼리 훅 구현
  - CSS 미디어 쿼리 문자열을 받아 현재 매칭 여부를 boolean으로 반환하는 훅을 구현한다
  - `window.matchMedia` API를 사용하여 실시간 뷰포트 변화를 감지한다
  - SSR 환경에서는 기본값 `false`를 반환하여 하이드레이션 불일치를 방지한다
  - 리사이즈 이벤트 리스너를 정리(cleanup)하는 로직을 포함한다
  - _Requirements: 5.1, 5.4_

- [x] 2.2 대시보드 상태 관리 훅 구현
  - 카테고리 선택, 검색어, 모바일 메뉴 토글 3개의 원시 상태를 관리한다
  - 선택된 카테고리로부터 페이지 제목을 파생한다 ("전체" -> "대시보드", 그 외 -> 카테고리명)
  - 선택된 카테고리와 검색어를 기반으로 목 스킬 데이터를 필터링하는 파생 상태를 계산한다
  - 검색 필터링은 대소문자 무시, 스킬명/설명/태그에 대해 부분 일치를 수행한다
  - 미디어 쿼리 훅을 사용하여 768px 기준 모바일 여부를 판별한다
  - 데스크톱으로 전환 시 모바일 메뉴 열림 상태를 자동으로 닫는다
  - 상태 변경/액션 함수(카테고리 변경, 검색어 변경, 메뉴 토글/닫기)를 반환한다
  - _Requirements: 2.4, 2.5, 3.4, 5.1, 5.2, 5.3, 5.4_

- [x] 3. 말단 UI 컴포넌트 구현
- [x] 3.1 (P) 테마 전환 토글 버튼 구현
  - next-themes의 useTheme 훅을 사용하여 라이트/다크 모드를 전환하는 버튼을 구현한다
  - 현재 테마에 따라 Sun/Moon 아이콘을 교체 표시한다
  - 클릭 시 전체 레이아웃의 색상 테마를 즉시 전환한다
  - 사용자 선택 테마는 next-themes가 localStorage에 자동 저장하여 재방문 시 유지한다
  - 시스템 테마 설정(prefers-color-scheme) 감지 기능을 활용한다
  - 클라이언트 컴포넌트로 선언한다
  - _Requirements: 6.1, 6.2, 6.3, 6.4_

- [x] 3.2 (P) 검색 입력 컴포넌트 구현
  - 검색 아이콘과 텍스트 입력 필드를 포함하는 검색 컴포넌트를 구현한다
  - 입력값 변경 시 상위 컴포넌트에 콜백으로 전달한다
  - 플레이스홀더 텍스트를 지원한다
  - _Requirements: 3.2_

- [x] 3.3 (P) 스킬 카드 컴포넌트 구현
  - 개별 스킬의 이름, 설명, 카테고리 태그를 카드 형태로 표시한다
  - 스킬 아이콘을 카드 상단에 표시한다
  - 다크/라이트 모드에서 적절한 색상을 표시한다 (기존 CSS 변수 활용)
  - _Requirements: 4.3_

- [x] 3.4 (P) 카테고리 내비게이션 컴포넌트 구현
  - 카테고리 정의 상수를 사용하여 "전체" 포함 6개 카테고리를 버튼 목록으로 렌더링한다
  - 각 카테고리에 lucide-react 아이콘을 함께 표시한다
  - 현재 선택된 카테고리에 활성(active) 스타일을 적용하여 시각적으로 구분한다
  - 클릭 시 상위 컴포넌트에 선택된 카테고리를 콜백으로 전달한다
  - Tab 키를 사용하여 모든 내비게이션 항목을 순서대로 탐색할 수 있도록 한다
  - 포커스된 항목에서 Enter 키를 누르면 해당 항목을 활성화한다
  - _Requirements: 2.2, 2.3, 2.4, 2.5, 7.4, 7.5_

- [x] 4. 복합 UI 컴포넌트 구현
- [x] 4.1 (P) 스킬 카드 그리드 컴포넌트 구현
  - 스킬 카드를 반응형 그리드로 배치한다 (모바일 1열, 태블릿 2열, 데스크톱 3열)
  - 표시할 스킬이 없을 경우 "등록된 스킬이 없습니다" 빈 상태 메시지를 표시한다
  - 화면 너비에 따라 그리드 열 수를 자동 조정한다
  - _Requirements: 4.2, 4.6, 5.5_

- [x] 4.2 (P) 사이드바 컴포넌트 구현

  - 상단에 플랫폼 로고 또는 서비스명("Eluo Skill Hub")을 표시한다
  - 카테고리 내비게이션 컴포넌트를 포함하여 5개 직군 카테고리를 렌더링한다
  - 데스크톱에서는 고정 너비로 화면 왼쪽에 고정 배치한다

  - z-index를 적절히 관리하여 오버레이 위에 사이드바가 표시되도록 한다

- [x] 4.3 (P) 상단 헤더 컴포넌트 구현
  - header 시맨틱 태그를 사용한다
  - 현재 선택된 카테고리명을 페이지 제목으로 표시한다
  - 카테고리가 변경되면 변경된 카테고리명을 페이지 제목에 반영한다
  - 검색 입력 컴포넌트를 포함한다
  - 오른쪽에 사용자 프로필 영역(아바타 플레이스홀더 아이콘)을 표시한다
  - 테마 전환 토글 버튼을 포함한다
  - 모바일(768px 미만)에서 햄버거 메뉴 버튼을 표시하고, 클릭 시 모바일 메뉴 토글 콜백을 호출한다
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 5.1, 6.2, 7.2_

- [x] 4.4 (P) 메인 콘텐츠 영역 컴포넌트 구현

  - "전체" 카테고리(기본/홈 상태)에서 대시보드 요약 정보(히어로 섹션)를 표시한다
  - 카테고리가 선택되면 해당 카테고리에 속하는 스킬 카드 목록을 그리드로 표시한다
  - 콘텐츠가 영역을 초과할 경우 세로 스크롤을 지원한다
  - 로딩 중 상태에서 로딩 인디케이터를 표시한다
  - 표시할 스킬 데이터가 없으면 빈 상태 메시지를 표시한다

- [x] 5. 레이아웃 셸 및 페이지 통합

  - 전체 뷰포트 높이(100vh)를 사용하여 스크롤 없이 레이아웃 골격을 표시한다
  - 사이드바를 화면 왼쪽에, 헤더를 사이드바 오른쪽 상단에, 메인 콘텐츠를 헤더 아래에 배치한다

  - 오버레이 외부 영역 클릭 시 모바일 사이드바를 닫는다
  - 데스크톱(768px 이상)에서 사이드바를 항상 표시한다
