# Feature Spec: 어드민 회원관리 페이지

- **Feature ID**: 006
- **Feature Name**: admin-user-management
- **Status**: Draft
- **Created**: 2026-03-02
- **Author**: Jaerin

---

## 1. 기능 개요 및 목적

`/admin/members` 경로에서 관리자가 가입된 회원 목록을 테이블 형태로 조회하고, 각 회원의 역할(role)을 드롭다운으로 변경할 수 있는 인터페이스를 제공한다.
또한 어드민 페이지 상단 헤더의 브레드크럼프가 현재 활성 탭(회원관리/스킬관리)을 동적으로 반영하도록 개선한다.

---

## 2. 유저 스토리

### 어드민 사용자

- **As a** 어드민 사용자로서,
  **I want** `/admin/members`에서 가입된 회원 전체 목록을 테이블로 보기를,
  **So that** 현재 가입된 회원 현황을 한눈에 파악할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 각 회원의 역할(role)을 드롭다운으로 변경하기를,
  **So that** 별도 페이지 이동 없이 빠르게 회원 권한을 조정할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 헤더 브레드크럼프에서 현재 어드민 탭 이름을 확인하기를,
  **So that** 지금 어느 관리 섹션에 있는지 맥락을 잃지 않고 작업할 수 있다.

---

## 3. 기능 요구사항

### 3.1 회원 목록 테이블

1. `/admin/members` 페이지는 가입된 모든 회원을 테이블(표) 형태로 표시한다.
2. 테이블은 최소 다음 컬럼을 포함한다: 이메일, 이름(없으면 "-" 표시), 역할(role), 가입일.
3. 데이터가 없을 경우 "등록된 회원이 없습니다" 와 같은 빈 상태 메시지를 표시한다.

### 3.2 역할 변경 드롭다운

4. 테이블의 역할(role) 컬럼에는 각 회원의 현재 역할이 드롭다운(Select) 형태로 표시된다.
5. 드롭다운 선택 가능한 역할 목록은 시스템에 정의된 역할 값으로 구성된다 (예: `user`, `admin`).
6. 드롭다운에서 역할을 변경하면 즉시 서버에 저장 요청이 전송된다.
7. 역할 변경 성공 시 성공 알림(Toast 또는 이에 준하는 UI)을 표시한다.
8. 역할 변경 실패 시 실패 알림과 오류 원인을 표시하고, 드롭다운 값은 변경 이전 상태로 롤백한다.
9. 자기 자신(현재 로그인한 어드민)의 역할 변경은 허용하지 않는다 (드롭다운 비활성화 또는 경고).

### 3.3 헤더 브레드크럼프 동적 반영

10. 어드민 페이지 상단 헤더의 브레드크럼프는 현재 활성 탭을 반영하여 동적으로 표시한다.
11. 브레드크럼프 구조: `어드민 > {현재 탭 이름}` (예: `어드민 > 회원관리`, `어드민 > 스킬관리`).
12. 탭 이동 시 브레드크럼프가 즉시 갱신된다.

---

## 4. 비기능 요구사항

- **보안**: 역할 변경 API는 `admin` 역할을 가진 사용자만 호출 가능하다 (서버 사이드 인가 검증).
- **성능**: 회원 목록 페이지 초기 로드 3초 이내 (constitution 성능 기준 준수).
- **접근성**: 드롭다운은 키보드 인터랙션을 지원한다.
- **UX**: 역할 변경 중(로딩 상태)에는 드롭다운이 비활성화되어 중복 요청을 방지한다.

---

## 5. 가정 사항

- 회원 데이터는 Supabase Auth의 `users` 테이블 및 `roles` 테이블에서 조회 가능하다.
- 시스템에서 사용 가능한 역할은 `user`, `admin` 두 가지이다.
- 어드민 레이아웃(사이드바 + 메인 콘텐츠 + 헤더)은 004-admin-page에서 이미 구현되어 있다.
- `/admin/members`는 004 스펙에서 정의한 '회원관리' 탭과 연결되는 라우트이다.
- 현재 헤더(`Header` 컴포넌트)의 브레드크럼프는 하드코딩되어 있으며, 이를 동적으로 변경하는 것은 이 스펙의 범위에 포함된다.

---

## 6. 제약 조건

- 라우트는 `/admin/members`로 고정한다.
- 역할 변경은 반드시 서버 사이드에서 인가 검증 후 처리한다.
- 회원 데이터 변경은 반드시 Aggregate Root를 통해 처리한다 (constitution DDD 원칙).
- `any` 타입 사용 금지 (constitution 원칙).
- 자기 자신의 역할 변경은 서버 사이드에서도 거부되어야 한다.

---

## 7. 수용 기준 체크리스트

- [ ] `/admin/members` 진입 시 가입된 회원 목록이 테이블 형태로 표시된다.
- [ ] 테이블에 이메일, 이름, 역할, 가입일 컬럼이 존재한다.
- [ ] 회원이 없을 경우 빈 상태 메시지가 표시된다.
- [ ] 역할 컬럼에 각 회원의 현재 역할이 드롭다운으로 표시된다.
- [ ] 드롭다운에서 역할 변경 후 즉시 서버에 저장 요청이 전송된다.
- [ ] 역할 변경 성공 시 성공 알림이 표시된다.
- [ ] 역할 변경 실패 시 실패 알림이 표시되고 드롭다운이 이전 값으로 롤백된다.
- [ ] 역할 변경 중 드롭다운이 비활성화된다.
- [ ] 현재 로그인한 어드민 자신의 역할 드롭다운은 변경 불가 상태로 표시된다.
- [ ] 역할 변경 API 호출 시 서버 사이드에서 admin 역할 검증이 수행된다.
- [ ] 헤더 브레드크럼프가 `어드민 > 회원관리` 형태로 표시된다.
- [ ] `/admin/skills` 탭으로 이동하면 브레드크럼프가 `어드민 > 스킬관리`로 갱신된다.
