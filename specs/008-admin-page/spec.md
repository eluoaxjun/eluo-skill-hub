# Feature Specification: 어드민 페이지

**Feature Branch**: `008-admin-page`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "어드민페이지 구현 - /admin 엔드포인트, role 기반 접근제어, 대시보드·회원관리·스킬관리·피드백관리 탭"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 관리자 대시보드 접근 (Priority: P1)

관리자(role: admin)가 `/admin`에 접속하면, 사이드바 네비게이션과 대시보드 화면이 표시된다. 대시보드에는 전체 회원 수, 전체 스킬 수, 누적 피드백 수 등 주요 지표가 요약 카드로 나타나고, 최근 등록된 스킬 목록과 최근 가입한 회원 목록이 함께 표시된다.

**Why this priority**: 관리자가 플랫폼 현황을 한눈에 파악하는 것이 어드민 페이지의 핵심 가치이며, 모든 관리 기능의 진입점이 된다.

**Independent Test**: 관리자 계정으로 로그인 후 `/admin`에 접속하여 대시보드 화면이 정상 렌더링되는지, 요약 지표와 목록이 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 인증된 관리자(role: admin)가 로그인한 상태, **When** `/admin`에 접속, **Then** 사이드바(대시보드, 회원 관리, 스킬 관리, 피드백 관리 메뉴)와 대시보드 메인 영역이 표시된다
2. **Given** 대시보드 페이지가 로드된 상태, **When** 화면이 표시됨, **Then** 전체 회원 수, 전체 스킬 수, 누적 피드백 수 요약 카드가 표시된다
3. **Given** 대시보드 페이지가 로드된 상태, **When** 화면이 표시됨, **Then** 최근 등록된 스킬 목록과 최근 가입한 회원 목록이 표시된다

---

### User Story 2 - 비관리자 접근 차단 및 안내 (Priority: P1)

일반 사용자(role: user)가 `/admin`에 접속하면, 관리자 전용 페이지임을 알리는 안내 메시지와 `/dashboard`로 이동하는 버튼이 표시된다. 비인증 사용자는 로그인 페이지로 리다이렉트된다.

**Why this priority**: 권한 기반 접근제어는 보안의 기본이며, 관리자 기능이 일반 사용자에게 노출되어서는 안 된다.

**Independent Test**: 일반 사용자 계정으로 `/admin`에 접속 시 안내 메시지와 대시보드 이동 버튼이 나타나는지, 비인증 상태로 접속 시 로그인 페이지로 리다이렉트되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 인증된 일반 사용자(role: user)가 로그인한 상태, **When** `/admin`에 접속, **Then** "관리자 전용 페이지입니다" 안내 메시지와 "대시보드로 이동" 버튼이 표시된다
2. **Given** 일반 사용자가 안내 페이지를 보고 있는 상태, **When** "대시보드로 이동" 버튼 클릭, **Then** `/dashboard` 페이지로 이동한다
3. **Given** 비인증(로그아웃) 상태, **When** `/admin`에 접속, **Then** 로그인 페이지로 리다이렉트된다

---

### User Story 3 - 탭 간 네비게이션 (Priority: P2)

관리자가 사이드바의 메뉴(대시보드, 회원 관리, 스킬 관리, 피드백 관리)를 클릭하면 해당 탭의 콘텐츠 영역으로 전환된다. 현재 선택된 탭은 시각적으로 강조 표시된다.

**Why this priority**: 각 관리 기능 간의 이동이 가능해야 관리자가 실질적인 업무를 수행할 수 있다.

**Independent Test**: 각 사이드바 메뉴를 클릭하여 해당 탭 콘텐츠가 전환되는지, 활성 탭이 시각적으로 강조되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 대시보드 탭을 보고 있는 상태, **When** 사이드바에서 "회원 관리" 클릭, **Then** 메인 영역이 회원 관리 콘텐츠로 전환되고, 사이드바에서 "회원 관리"가 활성 상태로 강조된다
2. **Given** 관리자가 임의의 탭을 보고 있는 상태, **When** 사이드바에서 다른 메뉴 클릭, **Then** URL이 `/admin/[탭명]`으로 변경되고 해당 탭의 콘텐츠가 표시된다

---

### User Story 4 - 회원 관리 (Priority: P2)

관리자가 "회원 관리" 탭에서 등록된 회원 목록을 조회할 수 있다. 회원 이름, 이메일, 역할(role), 가입일, 상태 정보를 테이블 형태로 확인한다.

**Why this priority**: 회원 관리는 플랫폼 운영의 핵심 관리 기능이다.

**Independent Test**: 회원 관리 탭 진입 시 회원 목록 테이블이 표시되고 각 항목의 정보가 올바르게 나타나는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 어드민 페이지에 있는 상태, **When** "회원 관리" 탭 클릭, **Then** 전체 회원 목록이 이름, 이메일, 역할, 가입일, 상태 컬럼이 포함된 테이블로 표시된다
2. **Given** 회원 목록이 표시된 상태, **When** 회원 수가 많을 경우, **Then** 페이지네이션 또는 무한스크롤로 목록을 탐색할 수 있다

---

### User Story 5 - 스킬 관리 (Priority: P2)

관리자가 "스킬 관리" 탭에서 등록된 스킬 목록을 조회할 수 있다. 스킬 이름, 카테고리, 등록일, 상태 등의 정보를 확인한다.

**Why this priority**: 스킬은 플랫폼의 핵심 콘텐츠이며 관리자가 스킬 현황을 파악해야 한다.

**Independent Test**: 스킬 관리 탭 진입 시 스킬 목록이 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 어드민 페이지에 있는 상태, **When** "스킬 관리" 탭 클릭, **Then** 전체 스킬 목록이 이름, 카테고리, 등록일, 상태 컬럼이 포함된 형태로 표시된다

---

### User Story 6 - 피드백 관리 (Priority: P3)

관리자가 "피드백 관리" 탭에서 사용자들이 남긴 피드백 목록을 조회할 수 있다. 피드백 내용, 작성자, 작성일, 대상 스킬 등의 정보를 확인한다.

**Why this priority**: 피드백 관리는 플랫폼 품질 개선에 중요하지만 대시보드 및 회원/스킬 관리보다 우선순위가 낮다.

**Independent Test**: 피드백 관리 탭 진입 시 피드백 목록이 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 어드민 페이지에 있는 상태, **When** "피드백 관리" 탭 클릭, **Then** 전체 피드백 목록이 내용, 작성자, 작성일, 대상 스킬 정보가 포함된 형태로 표시된다

---

### Edge Cases

- 관리자 role이 데이터베이스에서 변경되어 세션과 불일치할 경우, 다음 접속 시 최신 role 정보로 접근 권한을 재평가한다
- `/admin` 하위의 존재하지 않는 경로(예: `/admin/invalid`)에 접속 시, 대시보드로 리다이렉트하거나 404 안내를 표시한다
- 데이터가 아직 없는 초기 상태(회원 0명, 스킬 0건, 피드백 0건)에서도 대시보드가 정상적으로 표시되며 "데이터 없음" 상태를 안내한다
- 사이드바 네비게이션과 메인 콘텐츠 영역이 모바일/태블릿 화면에서도 사용 가능해야 한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 `/admin`으로 시작하는 URL을 어드민 페이지 영역으로 처리해야 한다
- **FR-002**: 시스템은 인증되지 않은 사용자가 `/admin`에 접근 시 로그인 페이지로 리다이렉트해야 한다
- **FR-003**: 시스템은 인증된 사용자의 role을 확인하여 admin이 아닌 경우 접근 거부 안내를 표시해야 한다
- **FR-004**: role이 user인 사용자에게는 "관리자 전용 페이지입니다" 안내 메시지와 `/dashboard` 이동 버튼을 제공해야 한다
- **FR-005**: 어드민 레이아웃은 좌측 사이드바 + 우측 메인 콘텐츠 구조여야 한다
- **FR-006**: 사이드바에는 대시보드, 회원 관리, 스킬 관리, 피드백 관리 4개 메뉴가 표시되어야 한다
- **FR-007**: 현재 활성화된 탭은 사이드바에서 시각적으로 구분되어야 한다 (강조 배경색, 텍스트 색상 변경)
- **FR-008**: 대시보드 탭은 전체 회원 수, 전체 스킬 수, 누적 피드백 수를 요약 카드로 표시해야 한다
- **FR-009**: 대시보드 탭은 최근 등록된 스킬 목록(최대 5건)을 표시해야 한다
- **FR-010**: 대시보드 탭은 최근 가입한 회원 목록(최대 5건)을 테이블 형태로 표시해야 한다
- **FR-011**: 회원 관리 탭은 전체 회원 목록을 이름, 이메일, 역할, 가입일, 상태 컬럼이 포함된 테이블로 표시해야 한다
- **FR-012**: 스킬 관리 탭은 전체 스킬 목록을 이름, 카테고리, 등록일, 상태가 포함된 형태로 표시해야 한다
- **FR-013**: 피드백 관리 탭은 전체 피드백 목록을 내용, 작성자, 작성일, 대상 스킬 정보가 포함된 형태로 표시해야 한다
- **FR-014**: 각 탭 전환 시 URL이 `/admin`, `/admin/members`, `/admin/skills`, `/admin/feedbacks` 형태로 변경되어야 한다
- **FR-015**: 사이드바 하단에 현재 로그인한 관리자의 프로필 정보(이름, 역할)가 표시되어야 한다
- **FR-016**: 헤더 영역에는 페이지 제목과 검색 입력란, 알림·설정 아이콘이 표시되어야 한다

### Key Entities

- **사용자 (User)**: 시스템에 가입한 사용자로, 이름(display_name), 이메일, 역할(role: admin/user), 가입일, 상태(active/pending) 속성을 가진다
- **스킬 (Skill)**: 플랫폼에 등록된 자동화 스킬로, 이름, 카테고리, 설명, 등록일, 상태 속성을 가진다
- **피드백 (Feedback)**: 사용자가 스킬에 대해 남긴 피드백으로, 내용, 작성자(사용자 연관), 대상 스킬(스킬 연관), 작성일 속성을 가진다

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 로그인 후 2초 이내에 대시보드 화면의 요약 지표를 확인할 수 있다
- **SC-002**: 비관리자(role: user)가 `/admin` 접속 시 1초 이내에 접근 거부 안내와 대시보드 이동 버튼을 확인할 수 있다
- **SC-003**: 관리자가 사이드바 메뉴 클릭 시 1초 이내에 해당 탭 콘텐츠로 전환된다
- **SC-004**: 비인증 사용자가 `/admin` 접속 시 즉시 로그인 페이지로 리다이렉트된다
- **SC-005**: 모든 관리 탭(대시보드, 회원 관리, 스킬 관리, 피드백 관리)이 정상적으로 데이터를 표시한다

## Assumptions

- 사용자의 role 정보는 기존 프로젝트의 인증 체계(Supabase Auth + profiles 테이블)에서 조회한다
- 레이아웃 스타일은 `stitch-html/admin-dashboard.html` 레퍼런스를 기반으로 한다 (글래스모피즘 사이드바, 네이비 프라이머리 색상, 노란 액센트 컬러)
- 회원 관리, 스킬 관리, 피드백 관리 탭의 세부 기능(생성, 수정, 삭제)은 본 피처의 범위에 포함하지 않으며, 목록 조회만 제공한다
- 대시보드의 "실시간 플랫폼 활동량" 섹션은 레퍼런스에 포함되어 있으나, 실시간 데이터 연동 없이 정적 표시로 구현한다
- 검색 기능과 알림 기능은 UI만 배치하며, 실제 동작은 후속 피처에서 구현한다
