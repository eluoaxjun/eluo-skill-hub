# Feature Specification: 스킬 삭제 기능

**Feature Branch**: `014-skill-delete`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "스킬 삭제 기능. 삭제버튼을 누르면 confirmDialog가 나타나야한다. 스킬 삭제를 하면 통계분석에도 영향을 미친다는 것을 알려줘야한다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스킬 삭제 확인 다이얼로그 (Priority: P1)

관리자가 스킬 카드의 "삭제" 버튼을 클릭하면, 삭제를 재확인하는 확인 다이얼로그가 나타난다. 다이얼로그에는 해당 스킬의 이름과 함께 삭제 시 통계 데이터에 영향을 미친다는 경고 메시지가 표시된다. 관리자는 "취소" 또는 "삭제" 버튼을 선택하여 삭제를 취소하거나 확정할 수 있다.

**Why this priority**: 삭제는 비가역적 작업이므로, 실수로 인한 데이터 손실을 방지하기 위해 확인 다이얼로그가 가장 먼저 구현되어야 한다.

**Independent Test**: 삭제 버튼 클릭 후 확인 다이얼로그가 올바르게 표시되고, 취소 시 아무 동작도 발생하지 않는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 목록 페이지에 있을 때, **When** 스킬 카드의 "삭제" 버튼을 클릭하면, **Then** 삭제 확인 다이얼로그가 나타나고, 해당 스킬의 이름이 표시된다.
2. **Given** 삭제 확인 다이얼로그가 열려 있을 때, **When** "취소" 버튼을 클릭하면, **Then** 다이얼로그가 닫히고 스킬은 그대로 유지된다.
3. **Given** 삭제 확인 다이얼로그가 열려 있을 때, **When** 다이얼로그 외부 영역(오버레이)을 클릭하면, **Then** 다이얼로그가 닫히고 스킬은 그대로 유지된다.

---

### User Story 2 - 통계 영향 경고 메시지 (Priority: P1)

관리자가 삭제 확인 다이얼로그를 볼 때, 해당 스킬과 연관된 통계 데이터(피드백 로그 등)가 함께 삭제되어 통계 분석 결과에 영향을 미칠 수 있다는 경고 메시지가 명확하게 표시된다.

**Why this priority**: 사용자 요구사항에 명시된 핵심 기능으로, 관리자가 삭제의 파급 영향을 충분히 인지한 상태에서 의사결정을 내릴 수 있도록 해야 한다.

**Independent Test**: 다이얼로그 내에 통계 영향 경고 문구가 포함되어 있는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 삭제 확인 다이얼로그가 열려 있을 때, **When** 관리자가 다이얼로그 내용을 확인하면, **Then** "이 스킬을 삭제하면 관련 피드백 및 통계 데이터도 함께 삭제되어 통계 분석에 영향을 미칩니다" 형태의 경고 메시지가 표시된다.
2. **Given** 삭제 확인 다이얼로그가 열려 있을 때, **When** 경고 메시지를 확인하면, **Then** 경고 메시지는 시각적으로 강조(경고 아이콘 또는 강조 색상)되어 관리자의 주의를 끈다.

---

### User Story 3 - 스킬 삭제 실행 (Priority: P1)

관리자가 삭제 확인 다이얼로그에서 "삭제" 버튼을 클릭하면, 해당 스킬과 관련된 모든 데이터(스킬 정보, 첨부 파일, 템플릿 파일, 피드백 로그)가 삭제되고, 스킬 목록이 갱신된다.

**Why this priority**: 확인 다이얼로그의 "삭제" 버튼이 실제로 삭제를 수행해야 기능이 완성된다.

**Independent Test**: 삭제 확인 후 스킬이 목록에서 사라지고, 관련 데이터가 모두 제거되었는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 삭제 확인 다이얼로그에서 경고를 확인한 관리자가, **When** "삭제" 버튼을 클릭하면, **Then** 스킬이 삭제되고 스킬 목록 페이지가 자동으로 갱신되어 삭제된 스킬이 표시되지 않는다.
2. **Given** 삭제가 진행 중일 때, **When** 삭제 요청이 처리되는 동안, **Then** "삭제" 버튼이 비활성화(로딩 상태)되어 중복 요청을 방지한다.
3. **Given** 삭제가 성공적으로 완료되었을 때, **When** 관리자가 결과를 확인하면, **Then** 삭제 성공 메시지(토스트 또는 알림)가 표시된다.
4. **Given** 삭제 과정에서 오류가 발생했을 때, **When** 시스템이 삭제에 실패하면, **Then** 오류 메시지가 표시되고 스킬 데이터는 그대로 유지된다.

---

### Edge Cases

- 삭제 요청 중 네트워크 오류가 발생하면 오류 메시지를 표시하고 스킬은 유지된다.
- 동시에 두 명의 관리자가 같은 스킬을 삭제하려고 하면, 먼저 처리된 요청만 성공하고 두 번째 요청은 "이미 삭제된 스킬" 오류를 표시한다.
- 삭제 대상 스킬에 첨부 파일이 없는 경우에도 정상 삭제되어야 한다.
- 삭제 버튼을 빠르게 여러 번 클릭해도 하나의 삭제 요청만 처리되어야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 스킬 카드의 "삭제" 버튼 클릭 시 확인 다이얼로그를 표시해야 한다(MUST).
- **FR-002**: 확인 다이얼로그에는 삭제 대상 스킬의 이름이 표시되어야 한다(MUST).
- **FR-003**: 확인 다이얼로그에는 스킬 삭제 시 관련 통계 데이터(피드백 로그 등)에 영향을 미친다는 경고 메시지가 포함되어야 한다(MUST).
- **FR-004**: 경고 메시지는 시각적으로 강조되어 관리자의 주의를 끌어야 한다(MUST).
- **FR-005**: 관리자가 "취소"를 선택하면 다이얼로그가 닫히고 아무 변경도 발생하지 않아야 한다(MUST).
- **FR-006**: 관리자가 "삭제"를 확인하면 해당 스킬의 모든 관련 데이터(스킬 정보, 마크다운 파일, 템플릿 파일, 피드백 로그)가 삭제되어야 한다(MUST).
- **FR-007**: 삭제 처리 중에는 "삭제" 버튼이 비활성화되어 중복 요청을 방지해야 한다(MUST).
- **FR-008**: 삭제 완료 후 스킬 목록이 자동으로 갱신되어야 한다(MUST).
- **FR-009**: 삭제 성공 시 성공 알림을, 실패 시 오류 알림을 표시해야 한다(MUST).
- **FR-010**: 삭제 기능은 관리자 권한이 있는 사용자만 사용할 수 있어야 한다(MUST).

### Key Entities

- **Skill**: 삭제 대상이 되는 핵심 엔터티. 제목, 설명, 아이콘, 카테고리, 상태, 마크다운 콘텐츠를 포함한다.
- **Skill Template**: 스킬에 첨부된 템플릿 파일. 스킬 삭제 시 함께 삭제되어야 한다.
- **Skill Feedback Log**: 사용자 피드백 데이터. 스킬 삭제 시 함께 삭제되며, 이로 인해 통계 분석 결과가 변경된다.
- **Storage File**: 스킬에 연결된 마크다운 파일과 템플릿 파일의 물리적 저장소 데이터.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 스킬 삭제를 3번의 클릭 이내(삭제 버튼 → 확인 다이얼로그 → 삭제 확인)로 완료할 수 있다.
- **SC-002**: 삭제 확인 다이얼로그에서 통계 영향 경고를 확인한 관리자의 100%가 경고 내용을 인지한 상태에서 삭제를 진행하거나 취소할 수 있다.
- **SC-003**: 삭제 작업은 5초 이내에 완료되어 관리자가 즉각적인 피드백을 받을 수 있다.
- **SC-004**: 삭제 후 관련 데이터(스킬, 템플릿, 피드백, 파일)가 100% 정리되어 고아 데이터가 남지 않는다.

## Assumptions

- 삭제는 소프트 삭제(soft delete)가 아닌 영구 삭제(hard delete)로 구현한다. 삭제된 스킬은 복구할 수 없다.
- 스킬 삭제 권한은 기존 관리자 권한 체계를 따른다.
- 삭제 시 관련 파일(마크다운, 템플릿)도 스토리지에서 함께 삭제된다.
- 피드백 로그 삭제는 데이터베이스의 CASCADE 또는 명시적 삭제로 처리한다.
