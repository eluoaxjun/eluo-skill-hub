# Feature Spec: 어드민 페이지

- **Feature ID**: 004
- **Feature Name**: admin-page
- **Status**: Draft
- **Created**: 2026-03-02
- **Author**: Jaerin

---

## 1. 기능 개요 및 목적

Eluo Skill Hub의 관리자 전용 대시보드를 제공한다.
`admin` 역할을 가진 사용자만 접근 가능하며, 회원과 스킬을 중앙에서 관리할 수 있는 인터페이스를 제공한다.

---

## 2. 유저 스토리

### 어드민 사용자
- **As a** 어드민 사용자로서,
  **I want** `/admin` 경로에 접근할 수 있기를,
  **So that** 회원과 스킬을 중앙에서 관리할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 사이드바에서 회원관리와 스킬관리 탭 간 이동을 할 수 있기를,
  **So that** 각 관리 섹션에 빠르게 접근할 수 있다.

### 일반 사용자
- **As a** admin 역할이 없는 사용자로서,
  **I want** `/admin` 경로에 접근했을 때 권한 없음 안내를 받기를,
  **So that** 잘못된 경로에 왔음을 인지하고 원래 페이지로 돌아갈 수 있다.

---

## 3. 기능 요구사항

### 3.1 접근 제어
1. `/admin` 라우터로 진입 시 현재 사용자의 역할(role)을 확인한다.
2. `roles` 테이블에서 사용자 역할이 `admin`인 경우에만 어드민 페이지에 접근을 허용한다.
3. 비로그인 사용자가 접근하면 로그인 페이지로 리다이렉트한다.
4. 로그인 상태이지만 `admin` 역할이 없는 사용자는 권한 없음(Unauthorized) 페이지를 표시한다.

### 3.2 권한 없음 페이지
5. 권한 없음 화면에는 "접근 권한이 없습니다" 또는 이에 준하는 메시지를 표시한다.
6. 이전 페이지(또는 홈)로 돌아가는 버튼을 제공한다.

### 3.3 대시보드 레이아웃
7. 어드민 페이지는 사이드바 + 메인 콘텐츠 영역으로 구성된 대시보드 레이아웃을 사용한다.
8. 사이드바는 어드민 전용 내비게이션을 제공하며, 현재 활성 탭을 시각적으로 구분한다.

### 3.4 사이드바 탭
9. 사이드바에는 다음 두 개의 탭이 존재한다:
   - **회원관리**: 가입된 사용자 목록 조회 및 관리
   - **스킬관리**: 등록된 스킬 목록 조회 및 관리
10. 각 탭은 해당 관리 섹션의 콘텐츠 영역으로 이동한다.

---

## 4. 비기능 요구사항

- **보안**: 역할 확인은 서버 사이드에서 수행하여 클라이언트 우회를 방지한다.
- **성능**: 어드민 페이지 초기 로드 3초 이내.
- **접근성**: 사이드바 탭은 키보드 내비게이션을 지원한다.

---

## 5. 가정 사항

- `roles` 테이블은 DB에 이미 구현되어 있으며, 사용자-역할 매핑 정보를 제공한다.
- 어드민 역할 값은 `'admin'` 문자열로 정의되어 있다.
- 인증은 Supabase Auth 기반으로 처리된다.
- 회원관리·스킬관리 탭의 세부 기능(CRUD 등)은 이번 스펙의 범위 밖이며, 별도 스펙으로 정의한다.

---

## 6. 제약 조건

- 어드민 페이지 라우터는 `/admin`을 사용한다.
- 역할 확인 로직은 클라이언트에서만 수행하면 안 되며, 서버 사이드 또는 미들웨어 레벨에서 검증해야 한다.
- `any` 타입 사용 금지 (constitution 원칙).

---

## 7. 수용 기준 체크리스트

- [ ] `admin` 역할 사용자가 `/admin`에 접근하면 대시보드 레이아웃이 표시된다.
- [ ] 비로그인 사용자가 `/admin`에 접근하면 로그인 페이지로 리다이렉트된다.
- [ ] `admin` 역할이 없는 로그인 사용자가 `/admin`에 접근하면 권한 없음 메시지가 표시된다.
- [ ] 권한 없음 화면에 돌아가기 버튼이 존재하고, 클릭 시 이전 페이지 또는 홈으로 이동한다.
- [ ] 사이드바에 **회원관리**, **스킬관리** 탭이 존재한다.
- [ ] 각 탭 클릭 시 해당 콘텐츠 영역으로 이동(또는 전환)한다.
- [ ] 현재 활성 탭이 시각적으로 구분된다.
- [ ] 역할 확인이 서버 사이드(미들웨어 또는 서버 컴포넌트)에서 이루어진다.
