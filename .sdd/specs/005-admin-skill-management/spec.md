# Feature Spec: 어드민 스킬관리 페이지

- **Feature ID**: 005
- **Feature Name**: admin-skill-management
- **Status**: Draft
- **Created**: 2026-03-02
- **Author**: Jaerin

---

## 1. 기능 개요 및 목적

`/admin/skills` 경로에서 관리자가 등록된 스킬을 조회하고, 새로운 스킬을 추가할 수 있는 관리 인터페이스를 제공한다.
스킬은 카테고리와 마크다운 파일로 구성되며, 등록된 스킬 클릭 시 마크다운을 시각적으로 도식화된 형태로 미리볼 수 있다.

---

## 2. 유저 스토리

### 어드민 사용자
- **As a** 어드민 사용자로서,
  **I want** `/admin/skills`에서 등록된 스킬 목록을 테이블 형태로 보기를,
  **So that** 현재 등록된 스킬 현황을 한눈에 파악할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** '스킬 추가하기' 버튼을 눌러 새로운 스킬을 등록하기를,
  **So that** 마켓플레이스에 새 스킬을 공급할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 스킬 등록 시 카테고리를 선택하고 마크다운 파일을 업로드하기를,
  **So that** 스킬의 분류와 내용을 함께 등록할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 등록 성공/실패 시 즉시 알림을 받기를,
  **So that** 등록 결과를 바로 확인하고 필요 시 재시도할 수 있다.

- **As a** 어드민 사용자로서,
  **I want** 목록에서 스킬을 클릭하여 마크다운을 도식화된 형태로 미리보기를,
  **So that** 실제 사용자에게 어떻게 보이는지 확인할 수 있다.

---

## 3. 기능 요구사항

### 3.1 스킬 목록 테이블
1. `/admin/skills` 페이지는 등록된 스킬 전체를 테이블(표) 형태로 표시한다.
2. 테이블은 최소 다음 컬럼을 포함한다: 스킬명, 카테고리, 등록일, 상태(활성/비활성).
3. 테이블 행을 클릭하면 스킬 상세 미리보기 모달이 열린다.

### 3.2 스킬 추가 버튼
4. '스킬 추가하기' 버튼은 테이블 목록 뷰 위(상단)에 위치한다.
5. 버튼 클릭 시 스킬 추가 모달이 열린다.

### 3.3 스킬 추가 모달
6. 모달 내에 카테고리 선택(Select/Dropdown) UI가 존재한다.
7. 카테고리는 기존에 등록된 카테고리 목록에서 선택한다.
8. 모달 내에 마크다운 파일(.md)을 업로드할 수 있는 파일 업로드 영역이 존재한다.
9. 파일 업로드는 드래그 앤 드롭 또는 파일 선택 버튼을 통해 가능하다.
10. 업로드 가능한 파일 형식은 `.md` 파일로 제한한다.
11. 등록 버튼 클릭 시 카테고리와 마크다운 파일이 서버에 저장된다.
12. 카테고리 미선택 또는 파일 미업로드 상태에서는 등록 버튼이 비활성화되거나 유효성 검증 오류를 표시한다.

### 3.4 업로드 결과 알림
13. 스킬 등록 성공 시 성공 알림(Toast 또는 이에 준하는 UI)을 표시한다.
14. 스킬 등록 실패 시 실패 알림과 함께 오류 원인을 사용자에게 전달한다.
15. 알림은 일정 시간(예: 3초) 후 자동으로 사라지거나 사용자가 닫을 수 있어야 한다.

### 3.5 스킬 상세 미리보기 모달
16. 테이블에서 스킬 행을 클릭하면 해당 스킬의 상세 정보를 담은 모달이 열린다.
17. 모달에는 스킬의 메타 정보(스킬명, 카테고리, 등록일)와 마크다운 미리보기 섹션이 포함된다.
18. 마크다운 미리보기는 원시 텍스트가 아닌, 스타일이 적용된 도식화 형태로 렌더링한다.
19. 도식화 렌더링에는 다음 요소가 포함되어야 한다:
    - 제목(h1~h6) 계층 구분 스타일
    - 코드 블록 하이라이팅
    - 순서 있는/없는 목록 스타일
    - 표(table) 스타일
    - 인라인 코드, 강조(bold/italic) 스타일
    - 구분선(hr) 스타일

## 4. 비기능 요구사항

- **보안**: 스킬 추가 API는 `admin` 역할을 가진 사용자만 호출 가능하다 (서버 사이드 인가 검증).
- **성능**: 스킬 목록 페이지 초기 로드 3초 이내 (constitution 성능 기준 준수).
- **접근성**: 모달은 ESC 키로 닫을 수 있으며, 포커스 트랩을 적용한다.
- **파일 제한**: 업로드 가능한 마크다운 파일 크기는 1MB 이하로 제한한다.

---

## 5. 가정 사항

- 카테고리 데이터는 DB에 이미 존재하며, API를 통해 목록을 가져올 수 있다.
- 마크다운 파일의 원본은 Supabase Storage 또는 DB에 저장된다.
- 어드민 레이아웃(사이드바 + 메인 콘텐츠)은 004-admin-page에서 이미 구현되어 있다.
- `/admin/skills`는 004 스펙에서 정의한 '스킬관리' 탭과 연결되는 라우트이다.

---

## 6. 제약 조건

- 라우트는 `/admin/skills`로 고정한다.
- 마크다운 렌더링은 raw text 표시 금지 — 반드시 파싱하여 스타일이 적용된 HTML로 렌더링한다.
- `any` 타입 사용 금지 (constitution 원칙).
- 스킬 데이터 변경은 반드시 Aggregate Root를 통해 처리한다 (constitution DDD 원칙).
