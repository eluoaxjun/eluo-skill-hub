# Feature Specification: 어드민 스킬 추가 팝업

**Feature Branch**: `011-admin-skill-add-modal`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "스킬 추가 팝업 구현 - intercepting routes + parallel routes 조합의 모달, 임시저장 다이얼로그, published/drafted 상태 관리"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 스킬 추가 모달 열기 및 기본 정보 입력 (Priority: P1)

관리자가 스킬 관리 페이지에서 '새 스킬 추가하기' 버튼을 클릭하면, URL이 `/admin/skills/new`로 변경되면서 모달 형태의 팝업이 나타난다. 관리자는 모달 내에서 스킬의 기본 정보(이모지, 카테고리, 제목, 설명, 공개 여부)를 입력하고, 다운로드 가능한 템플릿 파일을 첨부하여 저장할 수 있다.

**Why this priority**: 스킬 등록의 핵심 흐름이며, 이 기능 없이는 새로운 스킬을 시스템에 추가할 수 없다.

**Independent Test**: '새 스킬 추가하기' 버튼 클릭 → 모달 표시 → 폼 필드 입력 → 저장 버튼 클릭으로 스킬이 정상 등록되는지 확인

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 관리 페이지(`/admin/skills`)에 있을 때, **When** '새 스킬 추가하기' 버튼을 클릭하면, **Then** URL이 `/admin/skills/new`로 변경되고 모달 팝업이 표시된다
2. **Given** 스킬 추가 모달이 열려있을 때, **When** 이모지 등록 버튼을 클릭하면, **Then** OS 기본 이모지 선택기가 나타난다
3. **Given** 스킬 추가 모달이 열려있을 때, **When** 모든 필수 필드(제목, 카테고리, 설명)를 입력하고 저장 버튼을 클릭하면, **Then** 스킬이 성공적으로 저장되고 모달이 닫히며 토스트 알림("스킬이 저장되었습니다")이 표시되고 스킬 목록이 자동 갱신된다
4. **Given** 스킬 추가 모달이 열려있을 때, **When** '공개 여부' 토글을 끄고 저장하면, **Then** 해당 스킬은 'drafted' 상태로 저장되어 사용자에게 노출되지 않는다
5. **Given** 스킬 추가 모달이 열려있을 때, **When** '공개 여부' 토글을 켜고 저장하면, **Then** 해당 스킬은 'published' 상태로 저장되어 사용자에게 즉시 노출된다

---

### User Story 2 - 파일 업로드 (설명 마크다운 + 템플릿 파일) (Priority: P2)

관리자가 스킬의 상세 설명을 마크다운 파일로 업로드하고, 사용자가 다운로드할 수 있는 템플릿 파일(.zip, .md)을 복수 첨부할 수 있다.

**Why this priority**: 스킬의 상세한 사용법 가이드 및 실행 가능한 템플릿을 제공하기 위한 핵심 기능이지만, 기본 정보 입력이 먼저 동작해야 한다.

**Independent Test**: 모달에서 설명 마크다운 및 템플릿 파일을 업로드하고 저장 후, 정상 저장되었는지 확인

**Acceptance Scenarios**:

1. **Given** 스킬 추가 모달이 열려있을 때, **When** 설명 마크다운 업로드 영역에 `.md` 파일을 첨부하면, **Then** 파일명이 업로드 영역에 표시된다
2. **Given** 설명 마크다운 파일이 첨부된 상태에서, **When** 저장 버튼을 클릭하면, **Then** 마크다운 내용이 스킬 데이터에 함께 저장된다
3. **Given** 설명 마크다운 파일이 첨부된 상태에서, **When** 파일을 제거하거나 다른 파일로 교체하면, **Then** 새로운 파일 정보로 업데이트된다
4. **Given** 스킬 추가 모달이 열려있을 때, **When** 템플릿 업로드 영역에 `.zip` 또는 `.md` 파일을 첨부하면, **Then** 파일명과 크기가 업로드 영역에 목록으로 표시된다
5. **Given** 템플릿 업로드 영역에서, **When** 여러 개의 파일을 추가하면, **Then** 모든 파일이 목록에 누적 표시된다
6. **Given** 100KB를 초과하는 템플릿 파일을 첨부하려 할 때, **When** 파일을 선택하면, **Then** 파일 크기 제한(100KB) 오류 메시지가 표시되고 해당 파일은 첨부되지 않는다
7. **Given** 템플릿 파일이 첨부된 상태에서, **When** 개별 파일의 삭제 버튼을 클릭하면, **Then** 해당 파일만 목록에서 제거된다

### User Story 3 - 모달 닫기 보호 및 임시저장 (Priority: P2)

관리자가 모달에서 내용을 입력한 상태에서: (1) 모달 외부 클릭은 무시된다, (2) '임시저장' 버튼 클릭 시 확인 다이얼로그를 통해 초안 저장이 가능하다, (3) X 버튼 클릭 시 데이터 손실 경고 다이얼로그가 표시된다.

**Why this priority**: 사용자 경험 보호를 위한 중요 기능으로, 의도치 않은 데이터 손실을 방지하고 작성 중 내용을 보존한다.

**Independent Test**: 모달 외부 클릭 무시 확인 → 임시저장 버튼 다이얼로그 동작 확인 → X 버튼 다이얼로그 동작 확인

**Acceptance Scenarios**:

1. **Given** 모달이 열려있는 상태에서, **When** 모달 외부 영역(오버레이)을 클릭하면, **Then** 클릭 이벤트가 무시되고 모달이 유지된다 (입력 상태와 무관하게 항상 무시)
2. **Given** 모달에서 하나 이상의 필드에 내용을 입력한 상태에서, **When** 닫기(X) 버튼을 클릭하면, **Then** '저장하지 않으면 입력한 내용이 사라집니다' 다이얼로그가 표시된다
3. **Given** X 버튼 다이얼로그가 표시된 상태에서, **When** '취소' 버튼을 클릭하면, **Then** 다이얼로그만 닫히고 모달로 돌아간다 (입력 내용 유지)
4. **Given** X 버튼 다이얼로그가 표시된 상태에서, **When** '닫기' 버튼을 클릭하면, **Then** 입력 내용을 버리고 모달이 닫힌다
5. **Given** 모달에서 아무 필드에도 입력하지 않은 상태에서, **When** 닫기(X) 버튼을 클릭하면, **Then** 다이얼로그 없이 모달이 즉시 닫힌다
6. **Given** 모달에서 '임시저장' 버튼을 클릭하면, **When** '입력한 내용을 초안으로 저장합니다' 다이얼로그가 표시된다, **Then** 다이얼로그에는 '취소'와 '임시저장' 버튼이 있다
7. **Given** 임시저장 다이얼로그가 표시된 상태에서, **When** '취소' 버튼을 클릭하면, **Then** 다이얼로그만 닫히고 모달로 돌아간다 (입력 내용 유지)
8. **Given** 임시저장 다이얼로그가 표시된 상태에서, **When** '임시저장' 버튼을 클릭하면, **Then** 현재 입력 내용이 필수 필드 유효성 검사 없이 'drafted' 상태로 저장되고 모달이 닫힌다

### User Story 4 - URL 직접 접근 시 전체 페이지 표시 (Priority: P3)

사용자가 `/admin/skills/new` URL을 직접 입력하여 접근하면, 모달이 아닌 전체 페이지로 스킬 추가 폼이 표시된다.

**Why this priority**: Intercepting routes 패턴의 기본 동작으로, 직접 URL 접근이나 새로고침 시 적절한 UX를 제공한다.

**Independent Test**: 브라우저 주소창에 `/admin/skills/new`를 직접 입력하여 전체 페이지 폼이 정상 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 사용자가 브라우저 주소창에 `/admin/skills/new`를 직접 입력할 때, **When** 페이지가 로드되면, **Then** 모달이 아닌 전체 페이지로 스킬 추가 폼이 표시된다
2. **Given** 스킬 추가 모달이 열린 상태에서, **When** 페이지를 새로고침하면, **Then** 전체 페이지 형태의 스킬 추가 폼이 표시된다

---

### Edge Cases

- 모든 필수 필드를 입력하지 않고 저장을 시도하면 어떤 필드가 비어있는지 유효성 검사 오류 메시지가 표시된다
- 이모지를 선택하지 않고 저장하면 기본 이모지 ⚡가 자동 적용된다 (이모지는 선택 사항)
- 마크다운 파일 외 다른 형식의 파일을 업로드하려 하면 `.md` 파일만 허용된다는 안내 메시지가 표시된다
- 네트워크 오류로 저장이 실패하면 오류 메시지가 표시되고 입력 내용은 유지된다
- 매우 긴 제목이나 설명을 입력하면 적절한 길이 제한이 적용된다
- 동일 제목의 스킬이 이미 존재해도 저장을 허용한다 (제목 유일성 제약 없음)
- 템플릿 파일로 `.zip`/`.md` 외 형식을 업로드하려 하면 허용 형식 안내 메시지가 표시된다
- 템플릿 파일이 100KB를 초과하면 파일 크기 제한 오류 메시지가 표시된다
- 템플릿 파일 없이 스킬을 저장할 수 있다 (템플릿은 선택 사항)
- 템플릿 파일이 이미 10개 첨부된 상태에서 추가 업로드를 시도하면 최대 개수 제한 오류 메시지가 표시된다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 스킬 관리 페이지에 '새 스킬 추가하기' 버튼을 제공해야 한다
- **FR-002**: '새 스킬 추가하기' 버튼 클릭 시, URL이 `/admin/skills/new`로 변경되면서 모달 형태의 팝업이 표시되어야 한다 (Intercepting Routes + Parallel Routes 조합)
- **FR-003**: 스킬 추가 모달은 다음 입력 필드를 포함해야 한다:
  - 이모지 등록 버튼 (OS 기본 이모지 선택기 연동)
  - 카테고리 선택
  - 제목 입력
  - 설명 입력
  - 마크다운 파일 업로드 영역 (설명용)
  - 템플릿 파일 업로드 영역 (다운로드 가능한 템플릿, `.zip`/`.md` 허용, 복수 파일 가능, 파일당 최대 100KB)
  - 공개 여부 토글 스위치
  - 임시저장 버튼
  - 저장 버튼
  - 상단 우측 닫기(X) 버튼
- **FR-004**: 시스템은 각 스킬의 공개 상태를 'published' 또는 'drafted'로 관리해야 한다
- **FR-005**: 공개 여부 토글이 켜진 상태로 저장하면 'published', 꺼진 상태로 저장하면 'drafted' 상태로 저장되어야 한다
- **FR-006**: 'drafted' 상태의 스킬은 일반 사용자에게 노출되지 않아야 한다. 단, 어드민 스킬 관리 목록에서는 drafted 스킬이 표시되며 관리자가 클릭하여 편집할 수 있어야 한다
- **FR-007**: 모달 외부 영역(오버레이) 클릭 시 클릭 이벤트를 무시하고 모달을 유지해야 한다 (입력 상태와 무관)
- **FR-008**: 모달에 '임시저장' 버튼을 제공해야 한다. 클릭 시 '입력한 내용을 초안으로 저장합니다' 메시지와 함께 '취소'/'임시저장' 버튼이 있는 확인 다이얼로그가 표시되어야 한다
- **FR-008-1**: 임시저장 다이얼로그에서 '취소'를 선택하면 다이얼로그만 닫히고 모달로 돌아가야 한다 (입력 내용 유지)
- **FR-008-2**: 임시저장 다이얼로그에서 '임시저장'을 선택하면 현재 입력 내용이 필수 필드 유효성 검사 없이 'drafted' 상태로 저장되고 모달이 닫혀야 한다
- **FR-009**: 저장하지 않은 입력이 있는 상태에서 닫기(X) 버튼 클릭 시 '저장하지 않으면 입력한 내용이 사라집니다' 메시지와 함께 '취소'/'닫기' 버튼이 있는 확인 다이얼로그가 표시되어야 한다
- **FR-009-1**: X 버튼 다이얼로그에서 '취소'를 선택하면 다이얼로그만 닫히고 모달로 돌아가야 한다 (입력 내용 유지)
- **FR-009-2**: X 버튼 다이얼로그에서 '닫기'를 선택하면 입력 내용을 버리고 모달이 닫혀야 한다
- **FR-009-3**: 아무 필드에도 입력하지 않은 상태에서 닫기(X) 버튼 클릭 시 다이얼로그 없이 모달이 즉시 닫혀야 한다
- **FR-010**: 설명 마크다운 파일(`.md`) 업로드를 지원해야 하며, 다른 형식의 파일은 거부해야 한다
- **FR-011**: 필수 입력 필드(제목, 카테고리, 설명) 미입력 시 유효성 검사 오류를 표시해야 한다
- **FR-014**: 다운로드 가능한 템플릿 파일 업로드 영역을 제공해야 한다. `.zip` 및 `.md` 형식만 허용한다
- **FR-015**: 템플릿 파일은 복수 파일 업로드가 가능해야 하며, 한 스킬당 최대 10개까지 첨부할 수 있다
- **FR-016**: 템플릿 파일의 개별 파일 크기는 100KB 이하로 제한해야 하며, 초과 시 오류 메시지를 표시해야 한다
- **FR-017**: 업로드된 템플릿 파일은 사용자가 스킬 상세 페이지에서 다운로드할 수 있어야 한다
- **FR-012**: `/admin/skills/new` URL에 직접 접근 시 전체 페이지 형태로 스킬 추가 폼이 표시되어야 한다
- **FR-013**: 공개 여부 토글의 기본값은 꺼짐(drafted)이어야 한다

### Key Entities

- **Skill**: 자동화 스킬을 나타내는 핵심 엔티티. 이모지 아이콘(icon, 미입력 시 기본값 ⚡), 카테고리, 제목, 설명, 마크다운 상세 설명, 공개 상태(published/drafted)를 속성으로 가진다. DB에 `icon` 필드 추가 필요 (현재 누락)
- **SkillTemplate**: 스킬에 첨부되는 다운로드용 템플릿 파일. 파일명, 파일 경로/URL, 파일 크기, 스킬 참조를 속성으로 가진다. 하나의 Skill에 여러 SkillTemplate이 연결될 수 있다 (1:N 관계). DB에 별도 테이블 또는 관계 구조 추가 필요
- **Category**: 스킬이 속하는 분류. 기존 카테고리 목록에서 선택한다

## Assumptions

- 카테고리 목록은 기존 시스템에 이미 정의되어 있으며, 스킬 추가 시 드롭다운으로 선택한다 (새 카테고리 인라인 생성 불가)
- 이모지 선택은 OS 네이티브 이모지 입력기를 활용한다 (별도 이모지 피커 라이브러리 불필요)
- 마크다운 파일 업로드 시 파일 크기 제한은 합리적인 기본값(1MB)을 적용한다
- 기존 스킬 테이블의 `status` 필드 값을 'active'/'inactive'에서 'published'/'drafted'로 DB 마이그레이션을 통해 변경한다. 기존 코드의 status 참조도 함께 업데이트한다
- 관리자 권한을 가진 사용자만 스킬 추가 모달에 접근할 수 있다 (기존 어드민 인증 체계 활용)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 스킬 기본 정보를 입력하고 저장하기까지 2분 이내에 완료할 수 있다
- **SC-002**: 모달 열기/닫기 시 페이지 전환 없이 부드러운 오버레이 전환이 이루어진다

- **SC-006**: 필수 필드 미입력 시 저장이 차단되고 사용자에게 명확한 안내가 제공된다
